## 📋 SVN 自动化与 USVN 集成

### USVN 数据库结构确认
- **数据库名**: `usvn`
- **核心表**:
  - `usvn_projects` - 项目表 (projects_id, projects_name, projects_description, projects_start_date)
  - `usvn_users` - 用户表 (users_id, users_login, users_password(MD5 哈希), users_email, users_creation_date)
  - `usvn_groups` - 用户组表 (groups_id, groups_name, groups_description)
  - `usvn_users_to_projects` - 用户 - 项目关联 (users_id, projects_id, access_level: 0=无/1=读取/2=写入/3=管理员)
  - `usvn_users_to_groups` - 用户 - 组关联 (users_id, groups_id)
  - `usvn_groups_to_projects` - 组 - 项目关联 (groups_id, projects_id, access_level)

### USVN 关键发现
- USVN 使用 MySQL 数据库管理用户和权限，**不使用** SVN 原生 `passwd` 文件
- 通过 `svnadmin create` 创建的仓库**不会自动出现在 USVN 数据库中**，需要手动插入 `usvn_projects` 表
- USVN 密码使用 **MD5 哈希**存储
- 公司已在用 USVN，不能迁移到原生 SVN，需要通过数据库操作实现自动化

### 自动化方案
- 通过 Flask API + MySQL 数据库操作实现 USVN 自动化
- 同时操作文件系统（svnadmin）和数据库（usvn_*表）
- 保持 USVN Web 界面供员工继续使用

## 🖥️ vSphere Automation API 7.0 研究

### 核心 API 端点（来自 Broadcom 官方文档）

| 功能 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 创建会话 | `/api/session` | POST | 返回 session_id |
| 查询 VM | `/api/vcenter/vm` | GET | 支持 `names.1=xxx` 参数做幂等检查 |
| VM 详情 | `/api/vcenter/vm/{vm}` | GET | 获取虚拟机详细信息 |
| 解析模板 | `/api/vcenter/vm-template` | GET | `names.1=模板名` 返回模板 ID |
| 部署 VM | `/api/vcenter/vm-template/library-items` | POST | JSON body 含 placement、datastore 等 |
| 查询数据存储 | `/api/vcenter/datastore` | GET | 选择可用空间最大的 |
| 解析资源池 | `/api/vcenter/resource-pool` | GET | `names.1=资源池名` |
| 查询主机 | `/api/vcenter/host` | GET | `filter.clusters.1=集群名` 与 ESXI_CLUSTER 配置对应 |
| 查询文件夹 | `/api/vcenter/folder` | GET | `filter.type=VIRTUAL_MACHINE` |

### 关键实现要点
1. **幂等检查**: 先带 `names` 参数查询 VM，失败则不带参数查询后本地匹配
2. **会话管理**: session_id 通过 header `vmware-api-session-id` 传递
3. **部署请求 JSON body**:
   ```json
   {
     "spec": {
       "name": "vm-name",
       "template": "vm-template-id",
       "placement": {
         "resource_pool": "resgroup-xxx",
         "folder": "group-vxxx",
         "host": "host-xxx",
         "datastore": "datastore-xxx"
       },
       "power_on": true
     }
   }
   ```
4. **认证**: Basic Auth (username@vsphere.local:password)

## 📝 其他技术记录

### SVN 目录结构
```
/var/www/html/src/files/svn/
└── <project>/
    └── conf/
        ├── passwd      # USVN 不使用，保持示例内容
        ├── authz       # USVN 动态生成或不用
        └── svnserve.conf
```

### MySQL 连接命令
```bash
mysql -u root -p usvn
# 或
mysql -u usvn -p usvn
```

### htpasswd 命令说明
```bash
# 创建第一个用户（-c 创建文件）
htpasswd -cb passwd admin admin123

# 添加/更新用户（不加-c）
htpasswd -b passwd user password

# 删除用户
htpasswd -D passwd user
```

## 🔐 安全注意事项
- USVN 密码使用 MD5 哈希（不安全，但需兼容）
- vSphere API 建议在生产环境启用 SSL 验证
- SVN 目录权限应设置为 `apache:apache`，权限 `775`
