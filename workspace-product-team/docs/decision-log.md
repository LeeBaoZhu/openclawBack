**业务价值**：直观展示 12 个 Cron 任务的执行状态和历史记录  
**目标用户**：系统管理员、产品团队

---

### 🔍 技术可行性评估（Architect）

#### 需求 1：系统日志可视化仪表盘

| 评估维度 | 评估结果 |
|----------|----------|
| **数据可用性** | ✅ 充足 |
| **技术复杂度** | 🟡 中等 |
| **推荐方案** | 基于现有 Control UI 扩展 |
| **预估工作量** | 3-5 人天 |

**数据源分析**：
- ✅ 现有数据：`memory/YYYY-MM-DD.md` 每日巡检报告（结构化）
- ✅ 日志位置：`~/.openclaw/workspace-sysadmin/memory/`（17 份历史报告）
- ✅ 数据格式：包含 warning/error/fatal 计数、问题分类、建议措施
- ⚠️ 缺失数据：历史趋势数据需从 memory 文件中解析提取

**技术方案推荐**：

```
┌─────────────────────────────────────────────────────────┐
│                    前端展示层                            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │ 健康度仪表盘 │  │ 告警趋势图   │  │ 问题分类占比    │  │
│  │ (红黄绿灯)   │  │ (7 天/30 天)  │  │ (饼图/柱状图)   │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    API 服务层                             │
│  ┌─────────────────────────────────────────────────────┐│
│  │ GET /api/health/dashboard  (聚合健康度数据)          ││
│  │ GET /api/health/trends     (历史趋势数据)           ││
│  │ GET /api/health/alerts     (告警列表)               ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    数据处理层                            │
│  ┌─────────────────────────────────────────────────────┐│
│  │ 解析 memory/*.md → 提取结构化数据 → 缓存到 SQLite    ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    数据源层                              │
│  memory/YYYY-MM-DD.md  (每日巡检报告)                   │
└─────────────────────────────────────────────────────────┘
```

**推荐技术栈**：
| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | React + Ant Design + ECharts | 与现有 Control UI 技术栈一致 |
| API | Node.js + Express | 与 OpenClaw Gateway 同语言 |
| 数据库 | SQLite (内嵌) | 无需额外部署，适合单机场景 |
| 部署 | 复用 Gateway 进程 | 无需独立服务，降低运维成本 |

**关键指标设计**：
- 健康度评分：100 分制（无 error=100，每个 error-10，每个 warning-2）
- 告警趋势：7 天/30 天 error/warning 数量折线图
- 问题分类：配置错误、权限问题、API 密钥缺失、网络问题等

#### 需求 2：Agent 任务执行看板

| 评估维度 | 评估结果 |
|----------|----------|
| **数据可用性** | ✅ 非常充足 |
| **技术复杂度** | 🟢 低 |
| **推荐方案** | 直接读取 cron/jobs.json |
| **预估工作量** | 1-2 人天 |

**数据源分析**：
- ✅ 数据位置：`~/.openclaw/cron/jobs.json`
- ✅ 数据结构：每个任务包含完整的 state 字段
  ```json
  {
    "state": {
      "nextRunAtMs": 1775498400000,
      "lastRunAtMs": 1775412000003,
      "lastRunStatus": "ok",
      "lastStatus": "ok",
      "lastDurationMs": 18060,
      "lastDeliveryStatus": "delivered",
      "consecutiveErrors": 0,
      "lastDelivered": true
    }
  }
  ```
- ✅ 实时性：任务执行后立即更新，无需额外采集
- ✅ 历史记录：`~/.openclaw/cron/runs/` 目录存储执行历史

**技术方案推荐**：

```
┌─────────────────────────────────────────────────────────┐
│                    前端展示层                            │
│  ┌─────────────────────────────────────────────────────┐│
│  │ 任务列表（表格）                                      ││
│  │ - 任务名称 | 状态 | 下次执行 | 连续错误 | 操作      ││
│  └─────────────────────────────────────────────────────┘│
│  ┌─────────────┐  ┌─────────────────────────────────┐   │
│  │ 执行成功率   │  │ 任务执行时间趋势（热力图）       │   │
│  │ (仪表盘)     │  │                                 │   │
│  └─────────────┘  └─────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    API 服务层                             │
│  ┌─────────────────────────────────────────────────────┐│
│  │ GET /api/cron/jobs         (任务列表)               ││
│  │ GET /api/cron/jobs/:id     (任务详情)               ││
│  │ POST /api/cron/jobs/:id/run(手动触发)               ││
│  │ PUT /api/cron/jobs/:id     (启用/禁用)              ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    数据源层                              │
│  cron/jobs.json (实时状态) + cron/runs/ (历史记录)      │
└─────────────────────────────────────────────────────────┘
```

**推荐技术栈**：
| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | React + Ant Design Table | 表格组件成熟，支持排序/筛选 |
| API | Node.js + Express | 与需求 1 复用 |
| 数据库 | 无需（直接读 JSON 文件） | 数据量小，直接读取即可 |
| 部署 | 复用 Gateway 进程 | 无需独立服务 |

**关键功能设计**：
- 状态标识：🟢 正常 / 🟡 警告 / 🔴 失败
- 筛选功能：按状态筛选（全部/正常/失败）
- 手动触发：支持立即执行指定任务
- 启用/禁用：支持临时禁用任务
