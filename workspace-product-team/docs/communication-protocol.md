# 团队通信协议

## 通信工具

使用 `sessions_send` 工具进行团队成员间通信。

## 通信格式

```json
{
  "tool": "sessions_send",
  "params": {
    "sessionKey": "<目标 agent sessionKey>",
    "message": "[通信类型] 通信内容"
  }
}
```

## 通信类型

### 1. 需求评审请求

**发起方**: ProductOwner  
**接收方**: Architect

```
[需求评审] 新功能 X 需要技术可行性评估
- 需求描述：...
- 业务价值：...
- 期望上线时间：...
- 请评估：技术可行性、工作量、风险
```

### 2. 技术方案反馈

**发起方**: Architect  
**接收方**: ProductOwner

```
[技术方案] 功能 X 的技术方案
- 方案 A: ... (优点/缺点/工作量)
- 方案 B: ... (优点/缺点/工作量)
- 推荐：方案 A，原因...
- 请确认：是否按推荐方案执行
```

### 3. 风险预警

**发起方**: Architect  
**接收方**: ProductOwner

```
[风险预警] 发现技术风险
- 风险描述：...
- 影响范围：...
- 建议措施：...
- 紧急程度：高/中/低
```

### 4. 优先级变更

**发起方**: ProductOwner  
**接收方**: Architect

```
[优先级变更] 需求 Y 优先级调整
- 变更原因：...
- 新优先级：...
- 影响排期：...
- 请确认：是否影响当前迭代
```

### 5. 进度同步

**发起方**: 任意  
**接收方**: 另一方

```
[进度同步] 本周进展
- 完成：...
- 进行中：...
- 阻塞：...
- 下周计划：...
```

### 6. 决策请求

**发起方**: 任意  
**接收方**: 另一方

```
[决策请求] 需要共同决策
- 决策事项：...
- 背景信息：...
- 可选方案：...
- 建议：...
- 请讨论：...
```

## 通信记录

### 必须记录的通信

- 需求评审请求和结论
- 技术方案和选择
- 风险预警和处理
- 优先级变更和确认
- 重大决策和原因

### 记录位置

- **日常通信**: `memory/YYYY-MM-DD.md`
- **重大决策**: `docs/decision-log.md`

### 记录格式

```markdown
## [日期] 通信记录

### [时间] [通信类型]
- **发起方**: ProductOwner/Architect
- **接收方**: Architect/ProductOwner
- **内容**: ...
- **结论**: ...
```

## 响应时效

| 通信类型 | 期望响应时间 | 说明 |
|----------|--------------|------|
| 风险预警 (高) | 立即 | 阻塞性问题 |
| 需求评审 | 24h | 需要详细评估 |
| 技术方案 | 24h | 需要详细分析 |
| 优先级变更 | 12h | 影响排期 |
| 进度同步 | 24h | 常规同步 |
| 决策请求 | 24h | 需要讨论 |

## 通信最佳实践

1. **清晰主题** — 使用 [通信类型] 前缀
2. **完整信息** — 提供足够的背景和数据
3. **明确期望** — 说明需要什么响应
4. **记录结论** — 通信后记录结论到文档
5. **跟进闭环** — 重要事项跟进到完成
