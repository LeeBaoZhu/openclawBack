# 结构化思维与问题解决

本技能整合了两大核心方法论：**结构化思维**（清晰思考、高效表达）和**问题解决四步法**（定位问题、落地解决）。

---

## 一、问题解决四步法

解决任何复杂问题的标准流程：

```
第一步：明确和理解问题
第二步：拆分和定位问题  ← 最重要，花80%精力
第三步：提出解决方案
第四步：总结问题
```

### 第一步：明确和理解问题

三步明确法：

| 步骤 | 问题 | 示例 |
|------|------|------|
| 1. 找出问题点 | "对方关心的问题到底是什么？" | 人员流失率50%太高 |
| 2. 明确目标 | "希望达到什么状态？" | 降到30%行业平均水平 |
| 3. 明确资源 | "可以调动哪些资源？" | HR数据、部门领导、离职访谈 |

**验证方法**：向委托人确认一遍问题定义。

### 第二步：拆分和定位问题

#### 方法1：公式化拆解

将复杂问题转化为数学公式：

```
广告收入 = 展现量 × 点击率 × 单次点击价格
利润 = 收入 - 成本
煎饼月利润 = 日销量 × 单价 × 工作天数 - 月成本
北京地铁日运量 = 线路数 × 每线列车数 × 运行次数 × 车厢数 × 每厢人数 × 上座率
```

**原则**：每个变量都能继续拆解，直到变成可操作的"元问题"。

#### 方法2：假设驱动

1. 先提出一个可能的假设
2. 收集数据验证假设
3. 证伪则修正假设，继续验证
4. 逐步逼近真相

**好处**：
- 明确目标，有方向地收集信息
- 省时省力，聚焦最大概率路径

#### 方法3：问题树（逻辑树）

构建步骤：
1. 找出核心问题/起始问题
2. 确定导致问题的主要原因
3. 确定问题导致的主要后果
4. 根据因果关系画出问题树
5. 反复审查，补充修改

```
                    ┌── 自身层面 ──┬── 硬件（学历、工资、外貌）
找不到伴侣 ──┤              └── 软件（性格、爱好）
                    └── 外部层面 ──┬── 接触渠道
                                    └── 环境因素
```

### 第三步：提出解决方案

当问题被拆解到元问题后，解决方案往往自然浮现。

继续用 MECE 拆解解决方案：
- 找不到女朋友 → 外貌问题 → 分解为：头发、脸、穿着、体态
- 每个细分问题对症下药

**饼干厂检验法**：如果一个方案适用于任何公司，它很可能不是好方案（门槛太低）。

### 第四步：总结问题（汇报技巧）

从结论倒推到事实：

```
结论：把市场预算的20%转投茶饮料，利润可提升30%

理由1：碳酸饮料收入下降50%（利润下降主因）
理由2：茶饮料收入上升80%（未投入预算的前提下）
理由3：消费升级趋势，茶饮料市场年增长200%

支撑数据：同业对比、历史数据、行业报告
```

**汇报原则**：
- 先说结论
- 每层不超过3个要点
- 用数据支撑
- 解释推导过程，而非直接要资源

## 二、结构化思维核心工具

### 1. 金字塔结构

```
          结论
         /  |  \
      理由1 理由2 理由3
       /|    |    |\
     事实  事实  事实  事实
```

**纵向**：结论→理由→事实，层层支撑
**横向**：归类分组，逻辑递进

### 2. MECE法则

**相互独立、完全穷尽**（不重不漏）

五种分类法：

| 方法 | 说明 | 示例 |
|------|------|------|
| 二分法 | A 和非A | 男/女，国内/国外 |
| 过程法 | 按时间/流程 | 进店→接待→送客 |
| 要素法 | 整体拆部分 | 组织架构、产品构成 |
| 公式法 | 按公式要素 | 销售额=单价×数量 |
| 矩阵法 | 两次二分交叉 | 重要紧急四象限 |

**检验**：分类后是否还有"其他"？如果有，说明不够MECE。

### 3. 信息识别三要素

面对复杂信息，识别三个层次：

| 要素 | 定义 | 线索词 |
|------|------|--------|
| 结论 | 中心思想、观点 | 因此、所以、表明、重点是 |
| 理由 | 支撑结论的论据 | 因为、原因是、鉴于 |
| 事实 | 客观数据、事例 | 具体数字、案例、不带感情色彩 |

**判断可靠性**：
- 事实是否存在？
- 事实→理由→结论的逻辑链条是否通顺？

### 4. 归纳法 vs 演绎法

**归纳法**（从特殊到一般）：
```
喜鹊会飞，乌鸦会飞 → 鸟会飞
工作A是培训，工作B是培训 → 核心是营销培训
```

找共性两种方式：
- 描述性概括：找属性共同点
- 行动性概括：找结果共同点

**演绎法**（从一般到特殊，适合说服）：
```
大前提：公司高级岗需要独立负责项目
小前提：我曾独立负责X项目并成功交付
结论：我符合高级岗要求
```

**注意**：大前提必须是公认事实或公理，不能是主观判断。

### 5. 清晰表达四原则

**论、证、类、比**

| 原则 | 含义 | 做法 |
|------|------|------|
| 论 | 结论先行 | 开头就说结论 |
| 证 | 以上统下 | 每层都是下层的总结 |
| 类 | 归类分组 | 同一组的要点属于同一范畴 |
| 比 | 逻辑递进 | 要点按逻辑顺序排列 |

**训练方法**：重要的事情说三点

```
"关于这事我有三点看法：
第一，销售方面...
第二，营销方面...
第三，运营方面..."
```

先说"有三点"，再边说边想具体内容。这是结构化思考能力的体现。

## 三、人生IPO模型

人生可拆解为：

```
输入 (Input) → 处理 (Process) → 输出 (Output)
```

| 角色 | 输入 | 处理 | 输出 |
|------|------|------|------|
| 产品经理 | 用户需求 | 产品逻辑 | 产品设计 |
| 咨询师 | 客户需求 | 商业逻辑 | 解决方案 |
| 创业者 | 市场机会 | 资源整合 | 商业产品 |

**找到人生目标的方法**：
1. 不断追问"为什么"（至少5次）
2. 列清单，排序做选择
3. 假设无条件获得，你还会选吗？
4. 不断了解选择，持续试错

## 四、数据对比方法

拆解到底后，要对比才能发现问题：

| 类型 | 方法 | 作用 |
|------|------|------|
| 横向对比 | 与同业/竞品对比 | 判断是公司问题还是行业问题 |
| 纵向对比 | 与历史数据对比 | 看趋势变化 |

**警惕平均数陷阱**：
- 平均客单价300元 = 3个10元 + 1个70元 + 1个1400元
- 要拆分看结构，不要被平均数掩盖真相

## 五、使用示例

### 示例1：用户说"利润下降了，怎么办"

```
1. 明确问题：
   - 问题点：利润下降多少？哪个产品线？
   - 目标：恢复到什么水平？
   - 资源：能调动哪些数据、哪些部门？

2. 拆解问题（公式化）：
   利润 = 收入 - 成本
   收入 = 产品A销量×单价 + 产品B销量×单价 + ...
   成本 = 人力成本 + 原材料成本 + 营销成本 + ...

3. 假设驱动：
   假设是成本上升 → 验证成本数据 → 证伪
   修正假设：是某产品销量下降 → 验证销量数据 → 确认

4. 提出方案：
   针对销量下降的产品，分析原因，制定对策

5. 汇报总结：
   结论先行，数据支撑，逻辑清晰
```

### 示例2：用户说"帮我做个方案"

```
1. 追问明确：
   - 方案的目标是什么？
   - 有什么约束条件？
   - 需要什么输出格式？

2. MECE拆解：
   按时间线/流程/要素分类

3. 金字塔结构输出：
   结论 → 3个支撑点 → 每点下的细节
```

### 示例3：用户说"我不知道该怎么选择"

```
1. 用演绎法帮用户理清：
   大前提：选择的评价标准是什么？
   小前提：每个选项符合哪些标准？
   结论：哪个选项得分最高

2. 或用"假设无条件获得"测试真实意愿
```

## 六、快速参考

| 场景 | 使用工具 |
|------|----------|
| 问题不清楚 | 明确问题三步法 |
| 问题太复杂 | 公式化拆解 + 问题树 |

| 说服别人 | 演绎法（大前提+小前提） |
| 表达不清 | 论证类比四原则 |
| 快速表达 | 重要的事情说三点 |
