### REQ-002 - Agent 任务执行看板

**描述：** 直观展示 12 个 Cron 任务的执行状态和历史记录

**2026-05-11 拆分为两个 Phase：**
- ✅ **Phase 1 - 数据层**：plan.json 数据结构（Architect 已完成）
- ⏳ **Phase 2 - 可视化看板**：Web 界面开发（降级 P2，gui 仪表盘可部分替代）

**业务价值：** 
- 快速定位失败任务，减少排查时间
- 提升系统运维效率
- 降低监控门槛

**用户影响：** 系统管理员、产品团队（高频使用）

**验收标准（Phase 2）：**
- [ ] 任务列表展示（名称、状态、下次执行时间、连续错误次数）
- [ ] 状态筛选功能（全部/正常/失败）
- [ ] 手动触发任务执行
- [ ] 启用/禁用任务
- [ ] 执行成功率仪表盘

**技术备注：** 
- 数据源：`cron/jobs.json` + `cron/runs/`
- 技术方案：React + Ant Design Table，API 内嵌 Gateway
- 工作量：2.5 人天（Phase 2 剩余约 1.5 人天）
- 负责人：Architect（后端）、前端开发（前端）

**优先级：** P2（Phase 2 可视化）  
**状态：** Phase 1 已完成 / Phase 2 待排期

**提出日期：** 2026-04-24
