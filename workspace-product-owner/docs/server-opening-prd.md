## 一、需求背景

### 1.1 现状
公司已有自研项目管理系统（PHP+Vue），支持三种流程：**体验服**、**正式服**、**添加平台**。目前仅支持 RPG 类型游戏。

项目负责人创建项目后，可选择开通体验服，需填写：
- 项目英文名
- 开通的环境和平台
- GitLab 授权人员列表
- SVN 授权人员列表

**当前问题**：虽有开服接口，但 12 个部署步骤仍需人工处理，未实现端到端自动化。

### 1.2 体验服部署流程（12 个步骤）
| 序号 | 步骤 | 说明 |
|------|------|------|
| 1 | 初始化 GitLab 仓库 | 创建 server、proto、client 仓库 |
| 2 | 初始化 SVN 仓库 | 创建资源仓库 |
| 3 | 创建配置中心项目平台 | 配置中心项目创建 |
| 4 | 运营中心开通 | 开通运营后台权限 |
| 5 | 初始化项目角色权限 | 配置项目成员权限（含 bug 系统） |
| 6 | 部署虚拟机 | 分配计算资源 |
| 7 | 添加跳板机 | Jumpserver 访问配置 |
| 8 | 配置中心绑定实例 | 关联配置与实例 |
| 9 | 运营中心 Host 配置 | 域名/访问配置 |
| 10 | 获取 server 代码修改配置并提交 | 自动化配置代码 |
| 11 | 创建数据库 | 初始化数据库实例 |
| 12 | 修改并发布非游戏类配置 | 配置发布与复制 |

### 1.3 核心痛点
| 痛点 | 描述 | 影响 |
|------|------|------|
| 耗时久 | 从提交申请到服务可用平均耗时 1 天 | 研发等待周期不可控，影响开发效率 |
| 人工介入多 | 12 个步骤依赖人工操作 | 易出错，且无法保证一致性 |
| 入口分散 | 开服入口隐藏在多级菜单中 | 研发找不到发起入口，增加沟通成本 |

### 1.4 业务价值
- **效率提升**：缩短部署时间 50%，从 1 天压缩至半天
- **体验优化**：飞书对话式交互，降低使用门槛
- **研发聚焦**：研发只关注业务逻辑，环境配置由系统自动完成
- **战略意义**：
  - 为正式服自动化部署打基础（二期规划）
  - 建立统一服务入口模式，后续可扩展其他工具

## 二、目标用户

| 维度 | 描述 |
|------|------|
| 用户群体 | 技术负责人、项目负责人 |
| 用户规模 | 约 50 人 |
| 使用场景 | 新游戏项目启动时，从 0 部署可体验的 demo 环境 |
| 使用频率 | 按项目需求，预计每周 5-10 次 |
| 游戏类型 | 仅支持 RPG 类型（一期） |

## 三、产品目标

### 3.1 核心目标
1. **缩短部署时间 50%**：平均部署时长从 1 天压缩至半天
2. **一体化入口**：飞书统一对话入口，对话式完成体验服部署
3. **信息透明**：部署完成后自动推送完整配置信息给技术负责人和项目负责人
4. **自动化推进**：先实现部分自动化，逐步推进至全部自动化

### 3.2 成功指标
| 指标 | 当前值 | 目标值 | 衡量方式 |
|------|--------|--------|----------|
| 平均部署时长 | 1 天 | ≤半天 | 系统日志统计 |
| 人工介入步骤 | 12 步 | 逐步减少 | 流程日志统计 |
| 用户满意度 | - | ≥80% | 用户反馈调研 |
| 入口发现率 | - | ≥95% | 飞书消息触达率 |

## 四、功能需求

### 4.1 核心功能

#### 4.1.1 飞书对话入口
- **触发方式**：用户在飞书群聊/私聊中发送"开体验服"
- **交互流程**：
  1. 飞书机器人识别意图
  2. OpenClaw 收集必要信息
  3. 调用自研项目管理系统接口（已有接口，无文档）
  4. 返回部署结果通知

#### 4.1.2 信息收集
**需要用户输入的信息：**
| 字段 | 必填 | 说明 |
|------|------|------|
| 项目英文名 | 是 | 项目唯一标识 |
| 环境 | 是 | 部署环境选择 |
| 平台 | 是 | 目标平台 |
| GitLab 授权人员 | 是 | 有权限访问 GitLab 的人员列表 |
| SVN 授权人员 | 是 | 有权限访问 SVN 的人员列表 |

#### 4.1.3 部署结果通知
**通知对象**：技术负责人 + 项目负责人

**部署成功后推送以下信息：**
| 信息项 | 说明 |
|--------|------|
| Git 仓库地址 | server、proto、client 三个仓库地址 |
| SVN 仓库地址 | 资源仓库地址 |
| 开发机名字 | Jumpserver 中的开发机名称 |
| 数据库连接信息 | 数据库地址、账号、密码 |
| COS 桶配置 | 对象存储配置 |
| CDN 地址 | CDN 访问地址 |
| 项目所在目录 | 服务器上的项目路径 |
| 流水线地址 | 内部 CI/CD 系统地址 |
| 可访问服务地址 | IP + 端口，可直接访问体验 |

### 4.2 辅助功能

#### 4.2.1 状态查询
- 用户可查询部署进度
- 支持查询历史部署记录

#### 4.2.2 异常处理
- 部署失败时推送错误原因
- 提供人工介入通道（管理员联系方式）

### 4.3 功能范围边界

| 范围内（In Scope） | 范围外（Out of Scope） |
|-------------------|----------------------|
| 飞书对话式体验服部署 | 其他 IM 渠道接入 |
| RPG 类型游戏体验服 | 其他游戏类型支持 |
| 公司内部研发使用 | 外部用户访问 |
| 体验服部署 | 正式服部署（二期规划） |
| 部分自动化 | 全部自动化（逐步推进） |

## 五、非功能需求

### 5.1 性能要求
- 响应时间：飞书消息响应≤5 秒
- 部署处理时间：≤半天（依赖底层系统）

### 5.2 可用性
- 服务可用性：≥99%
- 支持并发：满足 50 人团队日常使用

### 5.3 安全性
- 权限控制：仅限公司内部人员使用
- 敏感信息：数据库连接信息等需脱敏/加密传输
- 访问控制：GitLab/SVN 授权人员需严格管控

### 5.4 可维护性
- 日志记录：完整记录 12 个部署步骤的执行情况
- 监控告警：部署失败时通知管理员
- 接口文档：补充自研系统 API 文档

## 六、技术方案概述

### 6.1 系统架构
```
用户 → 飞书机器人 → OpenClaw → 自研项目管理系统 → 执行 12 步部署流程 → 返回结果
```

### 6.2 技术选型
| 组件 | 技术方案 |
|------|---------|
| 对话入口 | 飞书机器人 |
| 意图识别/信息收集 | OpenClaw |
| 部署执行 | 自研项目管理系统（PHP+Vue，已有接口） |
| 通知推送 | 飞书消息卡片 |

### 6.3 依赖关系
| 依赖项 | 状态 | 说明 |
|--------|------|------|
| 飞书开放平台 | 需申请 | 机器人权限 |
| 自研系统 API 接口 | 已有 | 无文档，需对接确认 |
| OpenClaw 平台 | 已部署 | 无需额外配置 |
| GitLab | 已有 | 公司内网 |
| SVN | 已有 | 公司内网 |
| Jumpserver | 已有 | 跳板机系统 |
| 配置中心 | 已有 | 自研配置中心 |
| 运营中心 | 已有 | 自研运营后台 |

## 七、上线计划

### 7.1 时间节点
| 阶段 | 时间 | 交付物 |
|------|------|--------|
| 需求评审 | 第 1 周 | 需求文档定稿 |
| 接口对接 | 第 1 周 | API 接口规范确认 |
| 开发实现 | 第 1-2 周 | 功能开发完成 |
| 测试验收 | 第 2 周 | 测试报告 |
| 上线发布 | 第 2 周末 | 正式上线 |

### 7.2 资源投入
- 开发人力：1 人（全栈）
- 测试人力：1 人（可兼任）
- 产品人力：ProductOwner

### 7.3 风险与应对
| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 自研系统 API 无文档 | 高 | 提前与开发对接，梳理接口规范 |
| 飞书权限审批延迟 | 中 | 提前申请权限，并行开发 |
| 12 步流程部分无法自动化 | 中 | 优先自动化可自动化的步骤，人工步骤逐步优化 |
| 人力不足 | 中 | 优先保证核心功能，辅助功能延后 |
