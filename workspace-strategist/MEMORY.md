## 战略决策日志

### 2026-05-11 — 结构性障碍诊断：反思循环断裂根因

**问题**：Strategist 每周学习反思连续 5 周改进措施执行率 0%。

**根因诊断**：不是"不够主动"（行为问题），而是**结构性障碍**——Strategist 运行在 isolated cron session 中，只能写文件不能主动推送用户。"询问用户家和说状态"这类改进措施需要跨 session 通信能力，但当前未配置。

**发现的三个深层模式**：
1. **结构性 vs 行为性障碍区分**：连续 3 周 0% 执行率 → 大概率是工具/权限限制，非意愿问题
2. **反思产出收益递减**：5 周反思文档越来越长（6KB→19.5KB），执行率始终 0%，反思已变成"输出崇拜"
3. **战略文档僵尸化**：current.md（55天未更新）、okr.md（47天未更新），已从活文档变成死文档

**改进方向**：
- 为 Strategist 建立主动推送通道（sessions_send in cron）
- 引入"电路断路器"：同类改进措施连续 3 周不执行 → 停止提出该类措施，标记为结构性障碍
- 战略文档从"定期更新"改为"事件驱动更新"

**提交用户决策**：Strategist 是否需要主动推送能力？战略文档是否转为事件驱动维护？家和说项目继续/暂停/放弃？
