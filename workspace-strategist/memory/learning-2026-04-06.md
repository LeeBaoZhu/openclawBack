### 2. 【想】这些战略问题背后的模式是什么？

#### 模式一：知识生产 > 行动落地

**现象**：上周产出两个技能（财富自由5模块、学习力），但核心项目（家和说）未推进。

**深层原因**：
- 技能创建是「可控的输出」——我有完整控制权
- 项目推进涉及「外部执行」——需要用户行动
- 我偏向做我能做的事，而非用户需要的事

**模式提炼**：
```
知识型 Agent 倾向于：输出知识 > 推动行动
这导致：学习积累丰富，实践验证稀少
```

#### 模式二：待办事项 ≠ 执行动力

**现象**：待办事项写着"家和说体验版启动"、"建立周复盘习惯"，但没有实际推进。

**深层原因**：
- 待办事项只是「记录」，不是「驱动」
- 缺乏具体的触发机制和问责节点
- 没有将待办拆解为「下一步可做什么」

**模式提炼**：
```
战略文档中的待办事项 ≠ 可执行的行动
需要：触发机制 + 具体步骤 + 责任绑定
```

#### 模式三：学习闭环未形成

**现象**：创建了财富自由技能，但"尝试使用新创建的财富自由技能模块回答用户问题"未执行。

**深层原因**：
- 学习闭环 = 听 → 想 → 变 → 用
- 上周停留在「听 → 想 → 变」，「用」未发生
- 没有在创建时设定具体的应用场景和验证时间

**模式提炼**：
```
学习闭环断裂点：「用」的阶段最容易被跳过
原因：创建完成即产生「已完成」的错觉
```

### 3. 【变】如何优化规划方法？

#### 优化一：从「输出知识」转向「推动行动」

**改进思路**：
- 战略规划不只是写文档，要产出「下一步可做什么」
- 每个战略建议绑定具体的触发时机
- 减少纯知识输出，增加行动驱动

#### 优化二：待办事项升级为行动契约

**改进思路**：
- 每个待办事项必须有：
  - **触发机制**：什么时机启动
  - **具体步骤**：第一步做什么
  - **责任绑定**：谁负责、何时检查
- 不能只写"启动家和说"，要写"本周三 20:00 前完成家和说功能清单整理"

#### 优化三：强制学习闭环

**改进思路**：
- 创建技能/知识时，同步设定「用」的场景
- 在 MEMORY.md 记录时，必须回答「如何验证」
- 每周反思时检查「上周创建的东西用了没有」

## 可迁移的战略模式（提取）

### 模式1：知识-行动转换框架

```
战略输出 → 下一步可做什么 → 触发机制 → 责任绑定

使用场景：
- 当 Agent 倾向于输出知识而非推动行动时
- 当待办事项停留在"写下来"而非"做起来"时

价值：防止战略规划变成纯文档工作，确保落地执行
```

### 模式2：学习闭环强制卡点

```
听 → 想 → 变 → [强制卡点：用]
    创建时必须回答：
    1. 用在哪里？
    2. 何时验证？
    3. 怎么衡量效果？

使用场景：
- 当创建新技能/知识时
- 当学习成果停留在"写完"而非"应用"时

价值：防止学习闭环在「用」环节断裂
```

### 模式3：待办事项契约化模板

```
待办事项 = 
  任务描述 + 
  触发时机（何时开始） + 
  第一步行动（具体做什么） + 
  检查节点（何时确认）

示例：
❌ "家和说体验版启动"
✅ "本周三 20:00 前完成家和说功能清单整理，周五 10:00 检查进度"

使用场景：
- 当待办事项长期挂着不执行时
- 当战略文档缺乏执行力时

价值：将"愿望"转化为"契约"
```
