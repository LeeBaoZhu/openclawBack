## 2️⃣ 想 — 这些健康问题背后的模式

### 模式一：「系统化思维」外溢至健康管理
用户将产品经理的系统思维迁移到健康领域——从个人指标管理扩展至**全家庭成员的多维度健康运营**。这不是"看病"，而是"构建健康管理系统"。

### 模式二：「认知-执行鸿沟」是核心瓶颈
用户知道该做什么（早睡、运动、低脂），但习惯未完全对齐。这是健康顾问工作的**最大挑战**：如何帮助用户在认知和执行之间架桥。

### 模式三：「AI作为健康决策基础设施」
用户形成了稳定的 **"发现问题→AI咨询→线下行动"** 闭环。AI已成为家庭健康管理的默认第一触点——妈妈的健康问题、高血脂管理、四联疗程，都先问AI再行动。这要求健康指导**必须准确、可执行、附带就医指引**。

## 3️⃣ 变 — 如何优化健康指导方式

### 改进方向一：给行动清单，而非知识清单
用户不缺知识（知道该早睡运动），缺的是**可执行的微行动**。建议每个健康建议附带一个"本周可完成的最小行动"。

### 改进方向二：从"单次建议"升级为"持续追踪"
用户对体系化有强烈需求，建议：
- 健康建议附带"观察指标"（如：本周连续几天在0:30前入睡？）
- 下次交互时自动回顾上周行动达成情况

### 改进方向三：家庭健康的"分级响应"框架
妈妈的口腔问题、高血脂管理、备孕准备处于不同的**紧急程度**和**执行独立性**。建议按"紧急/重要"矩阵分类，让用户先看到优先级最高的行动项。

## 可迁移的健康管理模式

### 模式 A：「六层健康透视法」
从**个人→配偶→子女→父母→祖辈→生活方式**六个层次组织健康管理，而非单个疾病的应对。每个层次明确：
- 当前状态（正常/观察/干预中）
- 风险等级（低/中/高）
- 下次关注时间

### 模式 B：「认知-执行桥接公式」
**每次建议 = 健康知识（Why）+ 最小行动（What）+ 拆解障碍（How）**
示例："减少油炸" → "下周3次晚餐选蒸煮" → "如果加班吃外卖，选哪家可以避开油炸？"

### 模式 C：「故障即进化」→「健康即体系」
将产品管理的"每次故障产出改进需求"模式迁移到健康领域：
- 每次体检异常 → 产出跟踪指标 + 复查时间线
- 每次就医 → 检查家庭其他成员是否需要同步关注
- 每次治疗中断 → 完善应急预案
